# 挖掘机作业智能避障系统 — 领域上下文

## 系统概述

本系统（ExcavatorSafeNav）面向施工现场挖掘机的自动避障需求，通过多模态感知+分级决策实现安全作业。系统在 ROS Noetic + Gazebo 11 平台上验证，核心能力是在 ≤200ms 内识别威胁并触发对应的避障响应。

---

## 挖掘机作业模式

### walk（行进模式）
- **描述**：挖掘机沿规划路径移动，速度 0.5–1.5 m/s
- **避障重点**：路径前方动态障碍（行人、车辆），使用 RRT* 实时重规划
- **传感器激活**：摄像头全分辨率（640×480@30Hz）+ 激光雷达全扇区（360°@10Hz）
- **典型风险**：行人横穿（TTC < 2s），车辆并排（速差 > 1 m/s）

### rotate（回转模式）
- **描述**：上部回转体原地旋转，底盘不移动，角速度 ≤ 0.3 rad/s
- **避障重点**：回转半径内的静态障碍、近距离作业人员
- **传感器激活**：摄像头 + 激光雷达（侧向扇区权重加倍）
- **典型风险**：工人进入回转半径（安全距离 ≥ 3m），建材堆阻碍旋转

### dig（挖掘模式）
- **描述**：整机静止，动臂/斗杆/铲斗执行挖掘循环
- **避障重点**：施工人员意外进入工作范围（距离 < 2m）
- **传感器激活**：摄像头持续监测，激光雷达前方 90° 扇区
- **特殊说明**：挖掘模式下整机不移动，但铲斗运动轨迹产生动态遮挡，增加感知误报风险

## 施工现场安全约束

### 安全距离标准
| 障碍物类型 | 作业停止距离 | 警告距离 |
|----------|-----------|--------|
| 行人（person） | ≤ 1.0 m | ≤ 3.0 m |
| 车辆（vehicle） | ≤ 1.5 m | ≤ 4.0 m |
| 静态障碍（obstacle） | ≤ 1.0 m | ≤ 2.5 m |

*参考标准：JGJ 33-2012《建筑机械使用安全技术规程》第5.3.1条*

### 典型障碍物类型
- **行人**：工人、监理人员，速度 ≤ 1.5 m/s，热图特征明显
- **车辆**：运土卡车、混凝土泵车，速度 ≤ 10 km/h（场内限速）
- **静态障碍**：建材堆、钢筋笼、脚手架底座、工具车

### 施工环境约束
- 光照变化剧烈（直射阳光 → 阴影，亮度差可达 4 档 EV）
- 粉尘遮挡：PM2.5 > 500 μg/m³ 时激光雷达信噪比下降约 15%
- 地面不平：路面坡度 ≤ 8°（ISO 11374 要求）
- 同时作业设备 ≤ 3 台（本系统设计容量）

## 关键术语表

### TTC（Time-to-Collision，碰撞时间）
- **定义**：在当前相对速度保持不变的假设下，挖掘机与障碍物发生碰撞所需时间
- **公式**：TTC = distance / max(v_excavator − v_obstacle, ε)，ε = 0.01 防除零
- **意义**：TTC < 1.5s 为紧急停止触发阈值；TTC > 5s 为安全状态

### FSM（有限状态机，Finite State Machine）
- **定义**：系统安全决策的核心控制器，维护 NORMAL / CAUTION / PAUSED / EMERGENCY_STOP 四个互斥状态
- **转换触发**：由  中的连续  驱动，双阈值磁滞设计防止颤振
- **安全保证**：EMERGENCY_STOP 状态只能由人工  服务解除

### RRT*（Rapidly-exploring Random Tree Star）
- **定义**：渐近最优的随机采样路径规划算法，在 CAUTION 状态下为挖掘机计算绕行路径
- **改进点**：五次 Hermite 多项式路径平滑（C¹ 连续），输出曲率连续的可执行轨迹
- **参数**：step=0.5m，rewire=2.0m，max_iter=5000，目标容差=0.3m

### DeepSort（Deep Appearance SORT）
- **定义**：结合外观特征（ReID）与卡尔曼运动预测的多目标跟踪算法
- **作用**：为每个检测目标维护稳定 track_id，支持遮挡后 ID 复用
- **关键参数**：max_cosine_distance=0.2（外观相似度阈值），max_age=50（遮挡容忍帧数）

### GhostNetV2（幽灵网络 v2）
- **定义**：通过廉价线性变换生成特征图幽灵的轻量化主干网络
- **在本系统中的作用**：替换 YOLOv5 原始 CSP 主干，参数减少约 30%，在保持 mAP@0.5 的同时提升推理速度
- **验证结果**：GPU 640px 推理均值 13.3ms（CUDA 11.6，RTX 系列）

### risk_score（风险评分）
- **定义**：综合距离和 TTC 的归一化危险程度，范围 [0, 1]
- **公式**：risk_score = min(1, type_weight × (0.6 × dist_score + 0.4 × ttc_score))
- **阈值**：LOW < 0.3 ≤ MEDIUM < 0.7 ≤ HIGH
