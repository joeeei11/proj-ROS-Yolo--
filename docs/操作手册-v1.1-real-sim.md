## 一、系统当前能做什么

### 验证通过的三个场景

| 场景 | 结论 | 关键现象 |
|------|------|---------|
| **G1 headless 烟雾测试** | ✅ 通过 | 13个节点全部在线，激光雷达 10Hz，detected_obstacles ≥1Hz |
| **G2 行人场景** | ✅ 通过 | 行人靠近 → risk_level=2 → EMERGENCY_STOP → cmd_vel=0；手动 resume → NORMAL |
| **G3 静态绕障** | ✅ 通过 | 2026-05-12 以 simple 模型回归通过：`/planned_path` 为 `odom`，FSM=CAUTION，risk_level=1，模型姿态稳定 |

### 数据流（真实，无注入）

```
Gazebo 仿真世界
  ├─ 虚拟摄像头（640×480 @ 30Hz）→ YOLOv5(CPU) → DeepSort 跟踪
  └─ 虚拟激光雷达（360° @ 10Hz）  → lidar_processor → 聚类 + TF 坐标变换
                                              ↓
                              sensor_fusion（独立订阅 + 缓存融合）
                                              ↓
                              trajectory_predictor（卡尔曼速度 + TTC）
                                              ↓
                              risk_assessor → risk_state（LOW/MEDIUM/HIGH）
                                              ↓
                              FSM → 状态切换 → /cmd_vel
                                              ↓
                              RRT* planner → /planned_path + /planned_cmd_vel
```

### 三种 FSM 状态

| state 值 | 名称 | 触发 | cmd_vel |
|---------|------|------|---------|
| `0` | NORMAL | 风险 LOW | 正常行进 |
| `1` | CAUTION | 风险 MEDIUM（距离 < 5m） | 降速 0.7m/s |
| `3` | EMERGENCY_STOP | 风险 HIGH（距离 < 2.5m 或 TTC < 1.5s） | 0，锁定 |

## 四、启动方式

### 4.1 一键启动默认场景（主施工现场）

```bash
cd ~/excavator_ws && source devel/setup.bash
roslaunch excavator_gazebo full_simulation.launch
```

等待 **30~60 秒**，看到以下日志说明启动成功：

```
[ObstacleSpawner] 初始化完成
[SensorFusion] 节点启动
[FSMController] 初始状态: NORMAL
RRTStar: new path with N waypoints published
[monitor_server] 启动于 http://localhost:8080
```

### 4.2 无 GUI 模式（推荐，性能更稳定）

```bash
roslaunch excavator_gazebo full_simulation.launch headless:=true rviz:=false
```

### 4.3 场景 G2：行人靠近测试（推荐用此验证 EMERGENCY_STOP）

```bash
roslaunch excavator_gazebo full_simulation.launch \
  world_file:=$(rospack find excavator_gazebo)/worlds/test_scenarios/test_pedestrian.world \
  scenario:=pedestrian \
  rviz:=false
```

> ⚠️ 必须显式传 `world_file`，否则 launch 文件默认加载主场景。

### 4.4 场景 G3：静态障碍物绕行测试

```bash
roslaunch excavator_gazebo full_simulation.launch \
  world_file:=$(rospack find excavator_gazebo)/worlds/test_scenarios/test_static.world \
  scenario:=static \
  model_variant:=simple \
  rviz:=false
```

### 4.5 场景：高速车辆紧急停止

```bash
roslaunch excavator_gazebo full_simulation.launch \
  world_file:=$(rospack find excavator_gazebo)/worlds/test_scenarios/test_vehicle.world \
  scenario:=vehicle \
  rviz:=false
```

### 4.6 场景：多重威胁（最高等级决策）

```bash
roslaunch excavator_gazebo full_simulation.launch \
  world_file:=$(rospack find excavator_gazebo)/worlds/test_scenarios/test_multi_threat.world \
```

## 十二、已知问题与处理方法

### 问题 1：仿真启动后挖掘机立即飞离原点

**原因**：FSM + RRT* 从生成后立即开始规划行驶，若物理步长与质量设置不匹配，机器人会被弹飞或高速行驶。

**解决方法**：用传送服务将其拉回原点（见 §十一 步骤 3）。

---

### 问题 2：`/excavator/detected_obstacles` 无输出

**检查步骤**：

```bash
# 1. 确认 sensor_fusion 在线
rosnode info /sensor_fusion | grep Publications

# 2. 确认激光雷达有有效读数（不全为 inf）
timeout 5 rostopic echo /lidar/scan -n 1 | head -20

# 3. 恢复激光雷达（如果 Gazebo 物理卡死）
rosservice call /gazebo/pause_physics '{}' && sleep 1 && rosservice call /gazebo/unpause_physics '{}'
```

---

### 问题 3：`/lidar/scan` 全部 inf（无有效读数）

**原因**：Gazebo 物理引擎有时会卡死导致传感器停止发布。

**解决**：

```bash
rosservice call /gazebo/pause_physics '{}' && sleep 2 && rosservice call /gazebo/unpause_physics '{}'
```

---

### 问题 4：Port 8080 already in use

**原因**：上一次仿真没有完全清理干净。

**解决**：

```bash
~/kill_all.sh
```

或：

```bash
pkill -f monitor_server
```

---

### 问题 5：rosnode list 返回 `Unable to communicate with master`

**原因**：ROS Master 未启动或已退出。

**解决**：重新启动仿真，确保 `roslaunch` 已在运行。

```bash
~/kill_all.sh && ~/start_g3_simple.sh
```

---

### 问题 6：TF 坐标显示异常（world_x 为大负数）

**原因**：旧 ROS 会话的 robot_state_publisher 残留，导致 TF 帧偏移。

**解决**：

```bash
~/kill_all.sh
```

重新启动仿真即可（kill_all.sh 会清理所有 ROS 进程）。

## 十三、系统参数速查

### 关键 ROS 参数（运行时可修改）

```bash
# 激光雷达最小检测距离（当前 2.5m，过滤机身自遮挡）
rosparam get /lidar_processor/min_range

# 修改（立即生效，但节点需重启才实际应用）
rosparam set /lidar_processor/min_range 2.5

# 最小聚类点数（当前 1，允许远处稀疏障碍物被检测）
rosparam get /lidar_processor/min_cluster_points
```

```bash
# 风险阈值
rosparam get /risk_assessor/safe_distance      # 5.0m
rosparam get /risk_assessor/critical_distance   # 1.0m
```

### 话题与发布者速查

| 话题 | 发布者 | 频率 | 含义 |
|------|--------|------|------|
| `/lidar/scan` | Gazebo 激光雷达插件 | 10Hz | 原始 360° 扫描 |
| `/camera/image_raw` | Gazebo 摄像头插件 | 30Hz | 640×480 图像 |
| `/excavator/lidar_obstacles` | lidar_processor | 10Hz | 聚类结果（含 world 坐标） |
| `/excavator/tracked_obstacles` | deepsort_tracker | ~1Hz（CPU推理） | 视觉跟踪结果 |
| `/excavator/detected_obstacles` | sensor_fusion | ~10Hz | 融合输出（静态+视觉） |
| `/excavator/risk_state` | risk_assessor | ~10Hz | current_level/min_distance |
| `/excavator/system_state` | fsm_controller | ~10Hz | state(0/1/3) |
| `/planned_path` | rrt_star_planner | 按需 | RRT* 路径（nav_msgs/Path） |
| `/cmd_vel` | fsm_controller（唯一） | ~10Hz | 最终速度指令 |
