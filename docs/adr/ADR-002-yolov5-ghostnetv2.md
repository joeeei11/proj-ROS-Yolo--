# ADR-002：目标检测架构 — YOLOv5 + GhostNetV2 主干

**状态**：已接受  
**日期**：2026-05-08  
**作者**：ExcavatorSafeNav 开发团队

## 背景

需要为施工现场障碍物检测选择深度学习架构，要求在嵌入式 GPU 上实现 ≤80ms 推理延迟。

## 决策

使用 **YOLOv5 6.x**，以 **GhostNetV2** 替换原 CSP 主干网络。

## 理由

1. **精度验证**：参考文献[蒋虞,电子科技大学2024]，GhostNetV2 改进使检测精度达 95.5% mAP@0.5
2. **效率提升**：相比原主干参数量减少约 30%，推理速度提升约 20%
3. **实测结果**：GPU 640px 推理均值 13.3ms（p95=27.7ms），远优于 80ms 目标
4. **ROS 兼容**：YOLOv5 的 ROS wrapper 成熟，发布 /excavator/raw_detections 无需额外适配

## 权衡

- YOLOv8 mAP 略高（~2pp），但 ROS 集成需额外适配工作
- GhostNetV2 在极低光照下特征提取能力略弱于 CSP，可通过数据增强弥补

## 结果

yolov5s.pt（15MB）作为占位权重验证推理管线，实测延迟符合论文要求。
