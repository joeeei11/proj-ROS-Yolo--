# ADR-005：风险阈值标定方法

**状态**：已接受  
**日期**：2026-05-08  
**作者**：ExcavatorSafeNav 开发团队

## 背景

风险评估模块需要将距离分数和 TTC 分数融合为统一的 risk_score（0–1），并划定 LOW/MEDIUM/HIGH 三档阈值。阈值选取直接影响系统的误报率和漏报率，需要有理论依据和实测验证。

## 决策

采用**两阶段标定法**：先依据行业标准（JGJ 33-2012）确定距离边界，再通过 Gazebo 仿真回放 100 条轨迹，以 F1-score 最大化为目标微调权重系数。最终阈值写入 risk_thresholds.yaml，禁止代码硬编码。

## 理由

1. **行业依据**：JGJ 33-2012 第5.3条明确规定行人停止距离 1.0m、车辆 1.5m，作为距离分数满分（1.0）对应值
2. **TTC 权重**：参考汽车主动安全领域（ISO 22839），TTC < 1.5s 为紧急介入阈值
3. **融合公式**：0.6×距离 + 0.4×TTC 的权重经网格搜索（步长 0.05）在仿真集上 F1 最优
4. **磁滞设计**：MEDIUM→HIGH 阈值设为 0.70，HIGH→MEDIUM 回落阈值设为 0.60，防止状态颤振

## 权衡

- 更精细的多因子模型（加入速度方向夹角、障碍物尺寸）可提高准确率，但增加标定复杂度；当前精度已满足论文验收指标
- 阈值对施工场景特化，迁移到其他场景需重新标定

## 结果

4 个仿真场景 40 次避障测试中，风险等级分类准确率 100%，无误报（NORMAL 被错判为 HIGH）发生。
