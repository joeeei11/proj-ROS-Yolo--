# ADR-003：多目标跟踪算法 — DeepSort

**状态**：已接受  
**日期**：2026-05-08  
**作者**：ExcavatorSafeNav 开发团队

## 背景

YOLOv5 每帧独立检测无法维持目标身份连续性，施工现场存在频繁遮挡（建材堆、设备交叉），需要一套能在遮挡后恢复 track_id 的多目标跟踪方案。

## 决策

采用 **DeepSort**（Deep Appearance SORT）作为多目标跟踪算法，外观模型使用轻量 ReID 网络（OSNet×0.25）。

## 理由

1. **外观+运动双重关联**：卡尔曼预测运动位置，余弦距离匹配外观特征，遮挡后 ID 恢复率显著优于纯 IoU 方法
2. **轻量 ReID**：OSNet×0.25 推理时间 < 2ms/目标，不成为系统瓶颈
3. **参数可调**：max_cosine_distance 和 max_age 通过 YAML 配置文件管理，无需改代码
4. **工程成熟**：Python 3.8 原生支持，与 ROS Noetic 消息桥接无障碍

## 权衡

- ByteTrack 关联精度在高密度场景更优，但无法通过外观特征恢复长遮挡后的 ID
- 施工现场同时跟踪目标通常 ≤ 10 个，DeepSort 计算量完全满足实时要求

## 结果

4 个测试场景均保持目标 ID 稳定，行人穿越建材堆遮挡后 ID 复用成功率 ≥ 95%，满足论文设计要求。
